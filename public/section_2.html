On package.json and node modules

If we want to build complex applications with Node.js, simply running single JavaScript files with the node command won’t cut it. We need a way to use external packages and keep track of the dependencies in our project.

package.json

All npm packages contain a file, usually in the project root, called package.json - this file holds various metadata relevant to the project. This file is used to give information to npm that allows it to identify the project as well as handle the project's dependencies.

It can also contain other metadata such as a project description, the version of the project in a particular distribution, license information, even configuration data - all of which can be vital to both npm and to the end users of the package.

The package.json file is normally located at the root directory of a Node.js project.

Installing dependencies

So the first thing that needs to be created in a Node.js project is the package.json file, located in the root directory of said project. For example let’s say we want to build a project that uses Express.js (actually using Express.js will be covered in a further section) and need to install the dependency.

The express dependency has to appear in the package.json file and it would look something like this (version may vary):

                {
                    "dependencies": {
                        "express": "4.17.1"
                    }
                }

Simply typing this in the file will not add express to our project. We have to run a terminal command that will read our package.json file and install all dependencies: npm install

After running this command you will notice new files have appeared in your project directory.

package-lock.json

Firstly, package-lock.json is automatically generated for any operations where npm modifies either the node_modules tree, or package-lock.json. It describes the exact tree that was generated, such that subsequent installs are able to generate identical trees, regardless of intermediate dependency updates.

node_modules

Secondly, the node_modules is a directory only for build tools, the external modules that your project depends upon. When you npm install them, they are downloaded from the web and copied into the node_modules folder and Node.js is trained to look for them there when you import them (without a specific path)

To import external modules in Node.js we must use the built-in require function. For example, if we wanted to import the express module we installed earlier we would write:

                    const express = require("express");
